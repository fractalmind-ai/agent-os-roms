# OKR.md — Control Plane

## Structure Rules
- `OKR.md` only keeps **ACTIVE OKRs**.
- **At most 3 ACTIVE OKRs** at any time.
- **Each OKR has exactly 1 canonical tracker**.
- All **non-ACTIVE** (candidate / paused / complete) go to `okrs/Candidate.md`.
- Historical timelines / stale run ids / superseded context go to `okrs/archive/` and `memory/`.
